# Plan: Jira MCP Integration

**Date**: 2026-03-20
**Goal**: Add Jira (Atlassian hosted MCP) to Thor via the existing proxy, same pattern as Linear/PostHog.

## Context

Atlassian provides a hosted MCP server at `https://mcp.atlassian.com/v1/mcp` with Streamable HTTP transport. Auth is via Bearer token using a service account API key. This is identical to the Linear/PostHog pattern — no new container or custom server needed.

## Phases

### Phase 1 — Proxy config + docker-compose wiring

- Create `proxy.jira.json` with upstream URL, Bearer auth, and allowlisted tools
- Add port 3014 to proxy service in docker-compose.yml
- Add `JIRA_API_KEY` env var to proxy service
- Add `jira` MCP entry to `docker/opencode/opencode.json`
- Update proxy healthcheck to include port 3014
- Update Dockerfile EXPOSE for proxy target
- Add `JIRA_API_KEY` to `.env.example`

**Files to modify/create**:

| File                             | Action | Detail                                          |
| -------------------------------- | ------ | ----------------------------------------------- |
| `packages/proxy/proxy.jira.json` | Create | Upstream URL, auth header, tool allow/approve    |
| `docker-compose.yml`             | Edit   | Port 3014, env var, healthcheck                  |
| `docker/opencode/opencode.json`  | Edit   | Add jira MCP entry pointing to proxy:3014        |
| `Dockerfile`                     | Edit   | Add 3014 to EXPOSE                               |
| `.env.example`                   | Edit   | Add JIRA_API_KEY placeholder                     |

**Exit criteria**:

- Proxy starts on port 3014 with `proxy.jira.json`
- OpenCode can see Jira tools via the proxy
- `JIRA_API_KEY` flows from `.env` to proxy container

### Phase 2 — Verify connectivity + tune tool list

- Test that the proxy connects to `mcp.atlassian.com` and lists tools
- Verify read operations work (e.g., `getJiraIssue`, `searchJiraIssuesUsingJql`)
- Adjust allow/approve lists based on actual tool names returned by the upstream

**Exit criteria**:

- Proxy lists Jira tools from upstream
- At least one read operation succeeds through the proxy
- All proxy healthchecks pass

## Tool Access Policy

Based on the Atlassian MCP server's known tools:

**Allow (read operations)**:

| Tool                              | Purpose                    |
| --------------------------------- | -------------------------- |
| `getJiraIssue`                    | Read issue details         |
| `searchJiraIssuesUsingJql`        | Search issues via JQL      |
| `getVisibleJiraProjects`          | List accessible projects   |
| `getJiraProjectIssueTypesMetadata`| Issue type metadata        |
| `getJiraIssueTypeMetaWithFields`  | Field metadata for creation|
| `getTransitionsForJiraIssue`      | Available status transitions|
| `getJiraIssueRemoteIssueLinks`   | External links on issues   |
| `getIssueLinkTypes`              | Link type definitions      |
| `lookupJiraAccountId`            | Resolve user account IDs   |
| `atlassianUserInfo`              | Current user info          |
| `getAccessibleAtlassianResources`| Accessible sites/products  |
| `searchAtlassian`                | Cross-product search       |

**Approve (write operations)**:

| Tool                      | Purpose                    |
| ------------------------- | -------------------------- |
| `addCommentToJiraIssue`   | Comment on issues          |
| `createJiraIssue`         | Create new issues          |
| `editJiraIssue`           | Modify issue fields        |
| `transitionJiraIssue`     | Change issue status        |
| `createIssueLink`         | Link issues together       |
| `addWorklogToJiraIssue`   | Log work time              |

**Excluded (Confluence — out of scope)**:

All `*Confluence*` tools, `getPagesInConfluenceSpace`, `searchConfluenceUsingCql`, `updateConfluencePage`, `fetchAtlassian`.

## Decision Log

| #   | Decision                                          | Rationale                                                                                |
| --- | ------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| 1   | Use Atlassian hosted MCP (`mcp.atlassian.com`)    | Zero maintenance, same pattern as Linear/PostHog, official Streamable HTTP support        |
| 2   | Service account API key (Bearer token)            | Non-interactive auth, suitable for backend services, no OAuth flow needed                 |
| 3   | Jira-only to start, no Confluence                 | Focused scope; Confluence can be added later with its own proxy instance if needed        |
| 4   | All write tools behind approval                   | Consistent with existing policy; human approval for mutations                            |
| 5   | Port 3014 for proxy instance                      | Next available port after 3013 (grafana)                                                 |

## Out of Scope

- Confluence integration (can be added as a separate proxy instance later)
- Jira webhook triggers (would need gateway changes — separate plan)
- Custom Jira workflows or automation rules
