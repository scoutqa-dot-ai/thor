# Grafana MCP Integration — Loki, Tempo, and Prometheus

**Date**: 2026-03-16
**Status**: Complete

## Goal

Integrate the official [grafana/mcp-grafana](https://github.com/grafana/mcp-grafana) server
to give the AI agent access to **Loki** (logs), **Tempo** (traces), and **Prometheus-compatible**
metrics datasources such as Mimir.

## Architecture

```
OpenCode → Proxy (port 3015) → grafana-mcp (port 8000) → Grafana API → Prometheus / Loki / Tempo
```

- `grafana-mcp` runs as a Docker sidecar using the official `grafana/mcp-grafana` image
- Only `datasource`, `prometheus`, `loki`, and `proxied` (Tempo) tool categories enabled
- All other categories (dashboards, alerting, incidents, etc.) disabled
- Proxy enforces allow-list policy as with all other integrations

## Phases

### Phase 1 — Docker sidecar + proxy config

1. Add `grafana-mcp` service to `docker-compose.yml`
   - Image: `grafana/mcp-grafana:latest`
   - Transport: `streamable-http` on port 8000
   - Enabled tools: `datasource,prometheus,loki,proxied`
   - Env: `GRAFANA_URL`, `GRAFANA_SERVICE_ACCOUNT_TOKEN`
2. Create `packages/proxy/proxy.grafana.json` with allow-list
3. Add proxy instance on port 3015
4. Update `.env.example`

**Exit criteria**: `docker compose up --build` starts grafana-mcp, proxy connects, tools listed.

### Phase 2 — Verify & test

1. Verify tool listing via proxy health/tools endpoint
2. Test a Prometheus query, Loki query, and Tempo trace search
3. Confirm tools not in allow-list are hidden

**Exit criteria**: Agent can query Prometheus metrics, Loki logs, and Tempo traces.

## Decision Log

| #   | Decision                                             | Reason                                                                           |
| --- | ---------------------------------------------------- | -------------------------------------------------------------------------------- |
| 1   | Use official `grafana/mcp-grafana` Docker image      | Maintained by Grafana, full Prometheus + Loki + Tempo support                    |
| 2   | `streamable-http` transport                          | Matches proxy's `StreamableHTTPClientTransport` client                           |
| 3   | `--enabled-tools datasource,prometheus,loki,proxied` | Scope to datasource queries only; Tempo tools are "proxied" from Tempo's own MCP |
| 4   | Port 3015 for proxy                                  | Next in sequence after 3014 (git)                                                |

## Out of Scope

- Dashboard tools
- Alerting / incident tools
- ClickHouse / CloudWatch / Elasticsearch tools
