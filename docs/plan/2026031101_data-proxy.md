# Plan: Internal Data Proxy

**Goal**: Add a `data` container to Docker Compose that lets OpenCode access internal services via `curl http://data/service-name/path` without knowing API keys. The proxy maps service names to upstream URLs and injects per-service credentials.

**Why**: OpenCode needs to query internal APIs (e.g. Acme services) but we don't want API keys in its environment — keys leak through tool calls, logs, and session history. The data proxy holds credentials and appends them per-request.

## Architecture

```
OpenCode                          data proxy (nginx)                  Upstream
  |                                  |                                |
  |  GET http://data/acme-webapp/api/v1/bar                     |
  |----------------------------------------->|                        |
  |                                  |  GET https://app.acme.example/api/v1/bar
  |                                  |  + X-Admin-API-Key: <key>      |
  |                                  |------------------------------->|
  |                                  |                                |
  |          200 { ... }             |          200 { ... }           |
  |<---------------------------------|<-------------------------------|
```

### URL Mapping

Request to `http://data/<service>/<path>` → proxied to `<upstream>/<path>` with configured headers. The trailing `/` on `proxy_pass` strips the location prefix automatically.

### Configuration

Nginx template file (`docker/data/default.conf.template`) with env var substitution at container startup:

```nginx
location /acme-webapp/ {
    proxy_pass https://app.acme.example/;
    proxy_set_header X-Admin-API-Key ${ACME_WEBAPP_ADMIN_API_KEY};
    proxy_ssl_server_name on;
}

location /acme-agent/ {
    proxy_pass https://app.acme.example/;
    proxy_set_header X-Admin-API-Key ${ACME_AGENT_ADMIN_API_KEY};
    proxy_ssl_server_name on;
}
```

- `nginx:alpine` image runs `envsubst` on `*.template` files at startup
- Env vars set via Docker Compose environment
- Unknown paths → nginx default 404
- No code, no build step

## Phases

### Phase 1 — Nginx config + Docker Compose integration ✅

**Scope**: Nginx template config, Compose service, wire OpenCode dependency, test script.

**Files**:

- `docker/data/default.conf.template` — nginx config with env var placeholders
- `docker-compose.yml` — add `data` service (nginx:alpine, no ports exposed)
- `scripts/test-data-proxy.sh` — automated test

**Exit criteria**:

- `docker compose up data` starts and passes healthcheck
- No `ports:` mapping — not reachable from host
- Env vars interpolated into generated nginx config
- Known service routes proxy to upstream
- Unknown paths return 404
- Test script validates all of the above

### Phase 2 — Update OpenCode instructions to use http://data ✅

**Scope**: Add instructions to OpenCode's configuration so it knows to use `http://data/<service>/...` for internal API access.

**Exit criteria**:

- OpenCode instructions document available services (`acme-webapp`, `acme-agent`)
- Instructions explain the URL pattern: `curl http://data/<service>/<path>`
- OpenCode does not receive or see any API keys

## Decision Log

| #   | Decision                               | Reason                                                                                                                                                                       |
| --- | -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | ~~Express service~~ → nginx            | No code needed. nginx:alpine handles env var interpolation via templates, proxy_pass with trailing `/` strips prefixes, already used in stack for ingress. Zero maintenance. |
| 2   | Service name as first path segment     | Simple, no DNS tricks needed. `http://data/acme-webapp/api/foo` is clear and curl-friendly.                                                                                 |
| 3   | Port 80 (internal only, default nginx) | No `ports:` mapping in Compose — only reachable from other containers on the Docker network. Uses default port 80 since there's no host exposure.                            |

## Out of Scope

- Rate limiting
- Response caching
- Auth on the proxy itself (internal-only network)
- Dynamic config reload (restart to pick up changes)
- Request/response logging beyond health (add later if needed)
